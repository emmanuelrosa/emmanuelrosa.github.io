Learn how to do this or that in Grails

Hey, fellow coders. Join me and other Grails web app developers on Tue February 16 at 9 PM Eastern Standard Time to develop a simple Grails 3 web application, LIVE on Google Hangouts. It's FREE!

Sorry, you're too late. Registration for this hangout is closed.

This is not a peep show

If you think you're just gonna sit back, popcorn in hand, and watch someone code... forget about it! This is for coders who want to get their keyboards dirty. For coders willing to sweat over java.lang.OutOfMemoryError: PermGen space errors. And for coders who drop f-bombs when methods are being used outside of a Grails application.

Skills not included

Obviously, you must already have the ability to work with Grails. But you don't need to be a whiz. Having said that, we'd all benefit to have people with knowledge of...

  1. Twitter Bootstrap v3
  2. Grails Spring Security (to make the app multi-user)
  3. HTML
  4. CSS (probably mostly handled by Twitter Bootstrap)

via GIPHY

As for development tools, use what ever the hell you want. But the source code will be in a Mercurial repository hosted on Bitbucket. So you'll need basic Mercurial knowledge and a Bitbucket account.

How will this work?

I'll keep you posted on the details once you register, but the essence is this.

The requirements for the web app will be scoped out in advance. Each feature will have a ticket, so the work queue will be the tickets. I'll provide the Grails 3 project in the form of a Mercurial repository. The entire project, source code and tickets, will be publicly hosted on Bitbucket.

On the day of, we'll meet web-cam-to-web-cam in Google Hangouts. Don't worry gentlemen, you're not as ugly as you think. Then, we'll each grab the tickets we're interested in and start coding. You'll each create a named branch to work off of then merge and hg push --new-branch your changes to Bitbucket. You can always hg pull -u the latest code.

via GIPHY

As you encounter kinks, you'll have a group of coders to lean on. Just don't be lazy.

BTW, If you have jokes, especially of the corny variety, bring those too.

Grab your seat early

Due to the 10-user limit of Google Hangouts, seating is limited! So get in early. Besides, after a while, human beings just drive me nuts. So I can only handle a handful of you anyway.

FAQ

No one has asked any questions yet, so I'll make some up.

Are you just trying to get free labor?

Nope. You see, I don't have anyone near by to code with. It gets lonely. So I'm doing what any sane person would: seeking a romantic-ode relationship online.

How do I know you're not just trying to get free labor?

You don't. But if it makes you feel any better, we'll likely be coding a list web app. Yeah... think of those profit margins.

What do I get out of it?

It depends. You might learn a thing or two. Or you might learn nothing at all. If you bring some corny jokes with you, at the very least you'll get to witness the rest of us laugh at you hysterically.

Why can't we use GIT instead of Mercurial?

Because I can't stand GIT; Don't get me started. So basically, because I said so. If you know GIT and have never used Mercurial, this should help.
